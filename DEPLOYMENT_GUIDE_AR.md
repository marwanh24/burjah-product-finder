# دليل النشر خطوة بخطوة (للمبتدئين)

## الجزء 1: نشر الباك-إند على Render

### الخطوة 1 — احصل على مفتاح Salla API
1. ادخل إلى لوحة تحكم متجرك في سلة.
2. اذهب إلى: **الإعدادات → التطبيقات → مفاتيح API** (أو من خلال تطبيق Salla Partners / My Apps).
3. أنشئ Access Token جديد (Offline Access Token) بصلاحية قراءة المنتجات (`products.read`).
4. انسخ التوكن واحفظه في مكان آمن مؤقتاً (سنحتاجه في الخطوة 4).

### الخطوة 2 — أنشئ حساب Render (إن لم يكن لديك)
1. اذهب إلى https://render.com وأنشئ حساباً مجانياً (يمكنك الدخول عبر GitHub مباشرة).

### الخطوة 3 — اربط مستودع GitHub
1. من لوحة Render اضغط **New +** ثم **Web Service**.
2. اربط حساب GitHub الخاص بك واختر مستودع `gtm-server-render` (أو الاسم الذي رفعته عليه).
3. Render سيكتشف تلقائياً ملف `backend/render.yaml` الموجود في المشروع (Blueprint) — إن عرض عليك
   خيار "Apply Blueprint" اختره مباشرة، وإلا أدخل الإعدادات يدوياً كما يلي:
   - **Root Directory**: `backend`
   - **Runtime**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`

### الخطوة 4 — أضف مفتاح Salla كمتغيّر بيئة سرّي
1. في صفحة الخدمة على Render، اذهب إلى تبويب **Environment**.
2. أضف متغيّراً جديداً:
   - **Key**: `SALLA_API_TOKEN`
   - **Value**: التوكن الذي نسخته في الخطوة 1
3. تأكد أيضاً من وجود:
   - `ALLOWED_ORIGIN` = `https://burjah.com` (أو رابط متجرك الفعلي)
4. احفظ (Save Changes) — ستُعاد الخدمة تلقائياً بعد الحفظ.

### الخطوة 5 — انتظر النشر وانسخ الرابط
1. انتظر حتى تتحول حالة الخدمة إلى **Live** (دائرة خضراء).
2. انسخ رابط الخدمة الظاهر أعلى الصفحة، شكله مثل:
   `https://burjah-sync-backend.onrender.com`
3. للتأكد أن كل شيء يعمل، افتح في المتصفح:
   `https://burjah-sync-backend.onrender.com/health`
   يجب أن تظهر لك رسالة `{"ok":true, ...}`.
4. بعد دقيقة أو دقيقتين (أول مزامنة تعمل تلقائياً عند الإقلاع)، افتح:
   `https://burjah-sync-backend.onrender.com/live.json`
   يجب أن تظهر لك بيانات المنتجات (أسعار/مقاسات/توفر).

> **ملاحظة:** الخطة المجانية على Render قد "تنام" الخدمة بعد عدم استخدام؛ أول طلب بعدها يكون بطيئاً
> قليلاً (10-30 ثانية) ثم تعمل بسرعة عادية. إن أردت أداءً دائماً سريعاً، رقّي لخطة مدفوعة بسيطة (Starter).

## الجزء 2: تجهيز ولصق الويدجت في سلة

### الخطوة 1 — عدّل رابط الباك-إند داخل ملف الويدجت
1. افتح ملف `burjah-widget.html` (المرفق معك) في أي محرر نصوص (حتى Notepad يكفي).
2. ابحث (Ctrl+F) عن هذا السطر بالضبط:
   ```
   const LIVE_JSON_URL = 'https://YOUR-RENDER-SERVICE.onrender.com/live.json';
   ```
3. استبدل الرابط بين علامتي الاقتباس برابط خدمتك الفعلي من الجزء 1 (الخطوة 5)، مع إضافة
   `/live.json` في النهاية. مثال:
   ```
   const LIVE_JSON_URL = 'https://burjah-sync-backend.onrender.com/live.json';
   ```
4. احفظ الملف.

### الخطوة 2 — الصق الكود في محرر ثيم سلة
1. ادخل إلى لوحة تحكم سلة → **المتجر الإلكتروني → تصميم المتجر (الثيمات)**.
2. اضغط **تخصيص** أو **تعديل** على الثيم الحالي (Theme Editor / محرر الأكواد).
3. ابحث عن مكوّن باسم **"محتوى HTML"** أو **"HTML Component"** (يُضاف عادة كقسم داخل صفحة أو
   داخل الصفحة الرئيسية من قسم "إضافة عنصر/مكوّن").
4. أضف مكوّن "محتوى HTML" جديداً في المكان الذي تريد ظهور الأداة فيه (مثلاً أعلى الصفحة الرئيسية).
5. افتح ملف `burjah-widget.html` (بعد تعديله في الخطوة 1)، انسخ **محتواه بالكامل** (Ctrl+A ثم Ctrl+C).
6. الصقه داخل صندوق "محتوى HTML" في محرر سلة.
7. احفظ ونشر (Save / Publish).

### الخطوة 3 — جرّبي الأداة فعلياً
1. افتحي صفحة المتجر من متصفح عادي (وأيضاً من الجوال، فالتصميم يهتم بالجوال أولاً).
2. اضغطي "ابدئي الآن" وجاوبي على الأسئلة الست.
3. تأكدي أن النتائج تظهر بصور وأسعار ومقاسات حقيقية من متجرك.

### إذا ظهرت رسالة "تعذّر التحقق المباشر"
- تأكدي أن رابط `LIVE_JSON_URL` صحيح ومطابق تماماً لرابط Render + `/live.json` في النهاية.
- افتحي الرابط مباشرة في المتصفح وتأكدي أنه يعرض بيانات JSON وليس رسالة خطأ.
- تأكدي أن متغيّر `SALLA_API_TOKEN` في Render صحيح وأن الخدمة "Live" لا "Failed".

## تحديث البيانات لاحقاً
- الأسعار والمخزون تتحدّث تلقائياً كل 6 ساعات من نفسها — لا حاجة لأي تدخل يدوي.
- عند إضافة منتج جديد لأحد المسارات المدعومة، يجب تحديث ملف `data/products.json` (يتطلب مبرمجاً
  لتعديل `data/build-catalog.js` وإعادة توليده)، ثم إعادة رفع/بناء الويدجت من جديد.
